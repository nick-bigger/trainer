# Trainer

Nick's personal health system, in two parts:

1. A 75-Hard-style daily habit tracker web app (deployed to GitHub Pages, used from an iPhone
   as a standalone web app) that logs habits and morning weigh-ins to Turso.
2. A local-only AI nutritionist/strength coach CLI (`npm run coach`) that keeps its memories
   and the training plan as local files and reads the same remote Turso database.

The repo is public (GitHub Pages free tier). **`memory/` and `plan/` are gitignored and must
never be committed** - they hold personal health data and exist only on this machine. They are
not in git history; keep it that way.

## Stack

- Vite + React 19 + TypeScript, TanStack Router (file-based routes in `src/routes/`)
- Tailwind v4 + shadcn/ui primitives (`src/components/ui/`), single committed dark theme in
  `src/index.css`
- Turso (hosted SQLite) via `@libsql/client/web` HTTP transport from the browser
  (`src/lib/db.ts`) - habits, habit_logs, weights tables, seeded/migrated at startup
- Coach: `coach/cli.ts`, a terminal REPL on `@anthropic-ai/claude-agent-sdk`, run with
  `npm run coach` (one-shot mode: `npm run coach -- -p "<msg>"`). Uses the local Claude Code
  subscription credentials (no API key); `COACH_MODEL` overrides the model, `ANTHROPIC_API_KEY`
  switches to API billing if ever needed.
- Deployed to GitHub Pages via `.github/workflows/deploy.yml` on push to main.

## The coach's data (do not restructure casually)

- `plan/plan.md` - the living training/nutrition plan. The coach edits this file itself.
- `memory/MEMORY.md` + `memory/memories/*.md` - the coach's persistent memory (one fact per
  file, frontmatter with type + date). The index must stay one-line-per-memory.
- `.coach-session` (gitignored) - last Agent SDK session id, for conversation continuity.
- The tracking DB is the source of truth for daily habit checkoffs and weigh-ins; memory files
  are for everything else (DEXA scans, labs, PRs, workouts, preferences, injuries).
- The coach's permissions (`coach/cli.ts` canUseTool): read anywhere in the project, write
  only under `memory/` and `plan/`, DB access read-only via MCP tools, no Bash/web.
- These files exist only locally - they are covered by machine backups, not git.

## Env vars (`.env.local`, gitignored)

`VITE_TURSO_DB_URL`, `VITE_TURSO_AUTH_TOKEN` - used by both the browser client and the coach
CLI (see `.env.example`). The same values are GitHub Actions secrets for the Pages build.

## Gotchas

- `src/routeTree.gen.ts` is auto-generated by the router's Vite plugin but committed, because
  `tsc` runs before `vite build` and needs it to exist on a fresh checkout. Regenerate locally
  (`npm run dev` or `npm run build`) before committing route changes.
- Form text must render at 16px on mobile (`text-base md:text-sm`, never bare `text-sm`) or iOS
  Safari auto-zooms on focus.
- The app is used as an iPhone standalone web app: `viewport-fit=cover` plus safe-area handling
  (`env(safe-area-inset-top)` padding on each page header, `--bottom-nav-clearance` for the
  fixed nav + home indicator, `overscroll-behavior-y: none`). Keep new pages consistent.
- Habit completion is stored as row-presence in `habit_logs` (delete = uncheck), not a boolean
  column. The streak definition lives in `src/lib/tracking.ts` (`currentStreak`): today only
  counts once fully complete, and "fully complete" includes every active habit but NOT the
  weigh-in card.
- The seed habits in `src/lib/db.ts` use `INSERT OR IGNORE` - edits made to habit rows in the
  DB survive restarts; changing the seed array does not update existing rows.
